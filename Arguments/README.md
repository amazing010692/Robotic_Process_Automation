# Arguments — UiPath Argument Passing Demo

## Description

Demonstrates how to pass data between workflows using UiPath **Arguments** (In, Out, In/Out).

## How It Works

1. `Main.xaml` invokes `Another process.xaml`
2. Arguments are passed between the two workflows
3. Shows the difference between variables (local) and arguments (inter-workflow)

## Key Concepts

- **In Arguments** — Pass data into a sub-workflow
- **Out Arguments** — Return data from a sub-workflow
- **In/Out Arguments** — Bidirectional data flow

## How to Run

1. Open `project.json` in UiPath Studio
2. Run `Main.xaml`
